🚀 BACKEND INTEGRATION GUIDE FOR FRONTEND TEAM

This guide provides all the necessary information to integrate the EduCare Connect frontend with the FastAPI backend.

---

BASE CONFIGURATION

- Development Server URL: http://localhost:8000
- API Prefix: /api/v1
- Full API Base URL: http://localhost:8000/api/v1

---

AUTHENTICATION FLOW
The backend uses JWT (JSON Web Tokens) for authentication.

1. Login: Send a POST request to /api/v1/auth/login with 'username' (email) and 'password'.
   Note: This specific endpoint expects 'application/x-www-form-urlencoded' data.
2. Tokens: Upon successful login, you will receive an 'access_token' and a 'refresh_token'.
3. Authorized Requests: For all protected routes, include the 'access_token' in the Authorization header:
   Authorization: Bearer <your_access_token>
4. Refresh Token: Use /api/v1/auth/refresh when the access token expires.

---

API MODULES & ENDPOINTS

AUTH MODULE (/auth)
- POST /auth/register - Create a new user account.
- POST /auth/login - Authenticate and get tokens.
- POST /auth/refresh - Get a new access token using refresh token.
- GET /auth/me - Get current logged-in user details (Required Auth).
- GET /auth/verify-email - Verify email using token.
- POST /auth/forgot-password - Request password reset link.
- POST /auth/reset-password - Reset password using token.

EDUCATION MODULE (/education)
- GET /education/colleges - List all colleges.
- POST /education/colleges - Add a new college (Admin).
- GET /education/schools - List all schools.
- GET /education/coaching - List all coaching centers.
- GET /education/mess - List all mess facilities.

STAY MODULE (/stay)
- GET /stay/hostels - List all hostels.
- GET /stay/pg - List all PG accommodations.

MEDICAL MODULE (/medical)
- GET /medical/hospitals - List all hospitals.
- GET /medical/doctors - List all doctors.
- GET /medical/blood-banks - List all blood banks.
- GET /medical/ambulances - List all ambulances.

PROFILE & USER MODULE
- GET /profile/ - Get user profile details (Required Auth).
- PUT /profile/ - Update user profile (Required Auth).
- GET /users/ - List users (Admin).
- GET /settings/ - Get user settings (Required Auth).
- PUT /settings/ - Update user settings (Required Auth).

NOTIFICATIONS (/notifications)
- GET /notifications/ - Get list of notifications for user (Required Auth).
- PUT /notifications/{id}/read - Mark a notification as read (Required Auth).

INTEGRATION EXAMPLES (JAVASCRIPT)

1. Simple API Call with Fetch
async function getColleges() {
  const response = await fetch('http://localhost:8000/api/v1/education/colleges');
  return await response.json();
}

2. Login & Storing Token
async function login(email, password) {
  const formData = new URLSearchParams();
  formData.append('username', email);
  formData.append('password', password);

  const response = await fetch('http://localhost:8000/api/v1/auth/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: formData,
  });

  const data = await response.json();
  if (data.access_token) {
    localStorage.setItem('token', data.access_token);
  }
  return data;
}

3. Authenticated Request
async function getMyProfile() {
  const token = localStorage.getItem('token');
  const response = await fetch('http://localhost:8000/api/v1/profile/', {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
}
